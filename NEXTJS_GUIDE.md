# FinTrack Next.js & Supabase Web Setup & Vercel Deployment Guide

Welcome to **FinTrack Web**! Below is a comprehensive, production-grade guide detailing how to set up your Next.js Web Frontend and deploy it to **Vercel** to sync seamlessly with your existing Supabase database.

## Part 1: Supabase Database Setup

Before running either your **Android client** or **Next.js web client**, we must initialize the correct database tables on Supabase.

1. Go to your **Supabase Dashboard** (https://supabase.com).
2. Open your project: `kcodlflgjyvbwvvcxgjt`.
3. Locate the **SQL Editor** on the left menu (represented by a Terminal icon `>_`).
4. Select "+ New Query" and paste the following SQL schema to build your tables and enable API pathways:

```sql
-- Create expenses table
create table if not exists public.expenses (
    id bigint generated by default as identity primary key,
    amount double precision not null,
    category text not null,
    description text not null,
    date text not null, -- formatted as YYYY-MM-DD
    type text not null check (type in ('Expense', 'Income')),
    created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Create debts & loans table
create table if not exists public.debts (
    id bigint generated by default as identity primary key,
    name text not null,
    amount double precision not null, -- current outstanding balance
    limit_or_total double precision not null, -- total credit limit or original loan size
    interest_rate double precision not null, -- annual interest rate (e.g., 18.5)
    due_date text not null, -- description of due date
    type text not null check (type in ('Credit Card', 'Loan')),
    paid_amount double precision default 0.0 not null, -- amount paid off (for loans)
    created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Enable Row Level Security (RLS)
alter table public.expenses enable row level security;
alter table public.debts enable row level security;

-- Configure global select, insert, delete permissions for your Anon Key
create policy "Allow read on expenses" on public.expenses for select using (true);
create policy "Allow insert on expenses" on public.expenses for insert with check (true);
create policy "Allow delete on expenses" on public.expenses for delete using (true);

create policy "Allow read on debts" on public.debts for select using (true);
create policy "Allow insert on debts" on public.debts for insert with check (true);
create policy "Allow delete on debts" on public.debts for delete using (true);
```

5. Click **Run**. Your cloud backend is now online and available for connections!

## Part 3: Step-by-Step Vercel Deployment Instructions

Deploying your Next.js application to **Vercel** is extremely simple:

### Step 1: Push Web Repository to GitHub
1. Create a raw folder on your computer with the files detailed above.
2. Open terminal in that folder and initialize git:
   ```bash
   git init
   git add .
   git commit -m "Initialize fintrack nextjs system"
   ```
3. Go to GitHub and create a new **private or public repository** named `fintrack-web`.
4. Link it and push files:
   ```bash
   git remote add origin https://github.com/yourusername/fintrack-web.git
   git branch -M main
   git push -u origin main
   ```

### Step 2: Set up Vercel Deployment
1. Go to **Vercel** (https://vercel.com) and sign in using your GitHub account.
2. Click **Add New** -> **Project**.
3. Select your repository `fintrack-web` from the list and hit **Import**.
4. In the **Configure Project** stage:
   - Framework Preset: Leave as **Next.js**.
